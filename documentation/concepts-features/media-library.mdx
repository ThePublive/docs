## Change Image Globally

Replacing an asset in the Media Library updates it everywhere it is used across your content — posts, custom entities, and any other content types referencing that media item.

**Prerequisite:** Cache Tags must be enabled for your publisher. Without Cache Tags, the old image may remain cached on your CDN after replacement.

**Permission required:** Users must have the **Manage Media Library** permission to replace assets.

**How to replace an asset**

1. Open the asset in the Media Library.
2. Select **Replace** and upload the new file.
3. The asset URL stays the same — all content referencing this asset will serve the new file.

**Version History**

Each media item has a **Version History** tab showing all previous versions of that asset. This tab is read-only.

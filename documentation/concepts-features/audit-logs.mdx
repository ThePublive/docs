description: "Track who did what and when — a complete audit trail of dashboard actions across your workspace"
---

Audit Logs record all key dashboard actions with the acting member and a timestamp, giving you a complete activity trail across your workspace.

<Warning>
The Audit Logs UI is controlled by a feature flag. Contact [Publive support](https://help.thepublive.com/support/solutions) to enable the view for your publisher.
</Warning>

<Note>
Backend tracking is active for **all publishers** from the April 28, 2026 release onwards — logs are being captured even if the UI has not yet been enabled for your workspace.
</Note>

## What's logged

Each log entry records the **acting member** and a **timestamp**.

### Phase 1 — Content operations, team management, configuration

Tracked from **April 28, 2026**:

| Category | Actions tracked |
|----------|----------------|
| Content operations | Create, update, publish, and delete actions on content |
| Team management | Changes to team members and roles |
| Configuration | Changes to publisher settings and configuration |

### Phase 2 — Post-level events

Tracked from **May 19, 2026**:

| Event | Description |
|-------|-------------|
| Publish | A post is published |
| Unpublish | A post is unpublished |
| Schedule | A post is scheduled for future publication |
| Delete | A post is deleted |

## Accessing audit logs

The **Audit Logs** view is available in the Publive dashboard under your publisher settings. To enable it, contact [Publive support](https://help.thepublive.com/support/solutions).

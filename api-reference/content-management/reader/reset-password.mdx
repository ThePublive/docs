---
title: "Reset password"
description: "Set a new password using a reset token from email"
api: "PATCH https://cms.thepublive.com/publisher/{publisher_id}/reader/reset-password/"
---

Sets a new password for a reader using the one-time token sent by [Forgot password](/api-reference/content-management/reader/forgot-password). `new_password` and `confirm_password` are validated server-side — the request returns `400` if they do not match.

<ParamField path="publisher_id" type="string" required>
  Your Publisher ID
</ParamField>

<ParamField body="email" type="string" required>
  Reader's email address. Normalized to lowercase before forwarding.
</ParamField>

<ParamField body="token" type="string" required>
  One-time reset token from the password reset email.
</ParamField>

<ParamField body="new_password" type="string" required>
  The reader's new password.
</ParamField>

<ParamField body="confirm_password" type="string" required>
  Must match `new_password`. Validated server-side and not forwarded to the backend.
</ParamField>

<ResponseExample>

```json 200
{
  "status": "ok",
  "message": "user verification successful",
  "user_password_update": true,
  "forgot_password": true
}
```

```json 400 - Passwords do not match
{
  "status": "error",
  "message": "Passwords do not match."
}
```

```json 400 - Invalid or expired token
{
  "status": "error",
  "message": "User doesn't exists",
  "user_password_update": false
}
```

</ResponseExample>

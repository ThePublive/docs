---
title: "Login"
description: "Authenticate a reader and return a session token"
api: "POST https://cms.thepublive.com/publisher/{publisher_id}/reader/login/"
---

Authenticates a reader and returns an opaque session token. Supports two flows — email/password and OAuth (Google, Facebook) — on the same endpoint. The flow is determined by the presence of the `client` field.

<ParamField path="publisher_id" type="string" required>
  Your Publisher ID
</ParamField>

<Info>
  Use `Content-Type: application/json` for OAuth logins. For email/password, both `application/json` and form encoding are accepted.
</Info>

**Email / password flow**

<ParamField body="email" type="string" required>
  Reader's email address. Normalized to lowercase before forwarding.
</ParamField>

<ParamField body="password" type="string" required>
  Reader's password.
</ParamField>

<ParamField body="g-recaptcha-response" type="string">
  reCAPTCHA token from the client widget. Required when your publisher has a `login_captcha.secret_key` configured. There is no way to detect this requirement client-side — always include the token if you use a captcha widget.
</ParamField>

**OAuth flow** — include `client` to switch to this path

<ParamField body="client" type="string">
  OAuth provider. Accepted values: `google`, `facebook`. Presence of this field activates the OAuth flow.
</ParamField>

<ParamField body="email" type="string" required>
  Email from the OAuth provider. Normalized to lowercase.
</ParamField>

<AccordionGroup>
  <Accordion title="Google OAuth extra fields">
    | Field | Type | Description |
    | ----- | ---- | ----------- |
    | `sub` | string | Google subject identifier |
    | `name` | string | Full name |
    | `given_name` | string | First name |
    | `family_name` | string | Last name |
    | `picture` | string | Profile photo URL |
    | `email_verified` | boolean | Whether Google has verified the email |
    | `hd` | string | Hosted domain (G Suite accounts) |

    The full raw body is forwarded to the backend as JSON.
  </Accordion>
  <Accordion title="Facebook OAuth extra fields">
    | Field | Type | Description |
    | ----- | ---- | ----------- |
    | `id` | string | Facebook user ID |
    | `name` | string | Full name |
    | `first_name` | string | First name |
    | `last_name` | string | Last name |
    | `picture` | object | Profile picture object |

    The full raw body is forwarded to the backend as JSON.
  </Accordion>
</AccordionGroup>

<ResponseExample>

```json 200
{
  "status": "ok",
  "message": "User sign in success",
  "token": "abc123def456",
  "user_id": 42,
  "email": "alice@example.com"
}
```

```json 400 - Wrong password
{
  "status": "error",
  "message": "*Incorrect Password",
  "user_verification_successful": false
}
```

```json 400 - No account
{
  "status": "error",
  "message": "Invalid email or password",
  "user_verification_successful": false
}
```

```json 403 - Unverified email
{
  "status": "error",
  "message": "User is not verified",
  "user_verification_successful": false
}
```

```json 400 - Captcha failed
{
  "status": "error",
  "message": "Captcha verification failed"
}
```

</ResponseExample>

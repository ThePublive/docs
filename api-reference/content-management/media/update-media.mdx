---
title: "Update Media"
description: Update media asset metadata
api: "PATCH https://cms.thepublive.com/publisher/{publisher_id}/media-library/{id}/"
---

Updates metadata of an existing media asset. Only send the fields you want to change.

<Warning>
The following fields are **immutable after creation** and cannot be changed via update:
- `path` — Media URL / CDN path
- `type` — Media type (`Image`, `Video`, `File`)
</Warning>

#### Updatable fields

<ParamField path="publisher_id" type="string" required>
  Your Publisher ID
</ParamField>

<ParamField path="id" type="integer" required>
  Media ID
</ParamField>



<ParamField body="filename" type="string">
  Filename
</ParamField>

<ParamField body="alt_text" type="string">
  Alt text
</ParamField>

<ParamField body="caption" type="string">
  Caption
</ParamField>

<ParamField body="source" type="string">
  Source/credit
</ParamField>

<ParamField body="meta_data" type="object">
  Metadata
</ParamField>

#### Example request

```bash
curl -X PATCH \
  'https://cms.thepublive.com/publisher/123/media-library/999/' \
  -H 'Authorization: Basic YOUR_BASE64_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{"source": "Reuters", "alt_text": "Updated alt text"}'
```

<ResponseExample>

```json 200
{
  "status": "success",
  "message": "Updated Successfully",
  "data": {
    "id": 999,
    "filename": "budget-2026.jpg",
    "alt_text": "Updated alt text",
    "caption": "Budget session in Parliament",
    "path": "https://cdn.publive.com/images/budget-2026.jpg",
    "source": "Reuters",
    "type": "Image",
    "meta_data": {"width": 1200, "height": 630}
  }
}
```

```json 401
{
  "detail": "Invalid Auth Credentials"
}
```

</ResponseExample>
